Service name: Sales
Provides logic and limited process-centric services. Many of its method calls are simply routed directly through to the Data service (which is not accessed directly by any client software).
But most logic for total service time required, appropriate status to apply to a quote, and which items and quantities to send in supplier purchase orders is done within the Sales service.

URL: http://fastapps04.qut.edu.au/n5499879/Services/Sales/SalesService.svc
WSDL: http://fastapps04.qut.edu.au/n5499879/Services/Sales/SalesService.svc?wsdl

Customer[] GetCustomers()
Returns an array of all customers stored in the database

Customer GetCustomer(int customerID)
Returns the customer with the provided Customer ID

Quote[] GetQuotes()
Returns an array of all quotes stored in the database

Quote GetQuote(int quoteID)
Returns the quote with the provided Quote ID

int AddNewQuote(Quote quote)
Stores the provided quote in the database and returns its automatically generated Quote ID

void ModifyQuote(Quote quote)
Updates the provided quote in the database. Note: The quote being passed into the method must have a Quote ID that matches one stored in the database.

Item[] GetItems()
Returns an array of all items stored in the database

void ApproveQuote(int quoteID)
Updates status of the chosen quote to Approved. Depending on its previous status it will be set to one of the following: ApprovedCheckStock, ETAPendingApproval, ApprovedETA.

void RejectQuote(int quoteID)
Updates status of the chosen quote to Rejected. If a quote needs to be re-opened you can manually set the appropriate status using the ModifyQuote method.

void NotifyCustomer(int quoteID)
THIS METHOD IS NOT IMPLEMENTED: Sends a notification to the customer with the booking details.

DateTime GetPartETA(string partSKU)
Returns the Estimated Time of Arrival (date) for a part if an order is placed with the supplier on the current day.
This method communicates with the Supplier webservice.

Item[] CheckItemsStockLevel(Item[] items
Checks current stock levels in the warehouse for each part provided.
Also returns an array of items not in stock and the quantity required to order from supplier. This is handy for reporting back to the end user, but is not required to be stored by the client for placing orders with the supplier later on, this is done automatically by the Sales service if the ETA is approved by the customer.
